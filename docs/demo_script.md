# 答辩演示流程

## 1. 管理员演示

1. 访问 `http://localhost:8080/login`。
2. 使用 `admin / 123456` 登录。
3. 进入 `/admin/index`，展示学生总数、教师总数、课程总数、教学任务总数、成绩记录总数、待处理申诉数量。
4. 依次打开：
   - `/admin/students`
   - `/admin/teachers`
   - `/admin/courses`
   - `/admin/tasks`
5. 展示统计页面：
   - `/admin/stat/course`
   - `/admin/stat/class-credit`
   - `/admin/stat/region`
   - `/admin/stat/rank`

## 2. 教师录入和修改成绩

1. 退出管理员。
2. 使用 `t003 / 123456` 登录。
3. 打开 `/teacher/tasks`。
4. 选择 `TASK003`，进入选课学生页面。
5. 找到未录入成绩的学生，点击“录入”。
6. 输入平时成绩和期末成绩，提交。
7. 返回任务学生列表，查看最终成绩、等级、是否通过已由数据库触发器生成。
8. 点击“修改”，填写新平时成绩、新期末成绩和修改原因。
9. 提交后，打开管理员账号的 `/admin/audit/score`，展示成绩审计记录。

## 3. 发布成绩

1. 教师回到 `/teacher/tasks`。
2. 对 `TASK003` 点击“发布”。
3. 页面提示发布成功。

## 4. 学生查看成绩和 GPA

1. 退出教师。
2. 使用 `s2023011 / 123456` 登录。
3. 打开 `/student/scores`，展示已发布成绩。
4. 打开 `/student/gpa`，展示已修学分、GPA、通过课程数和未通过课程数。

## 5. 学生提交申诉

1. 在 `/student/scores` 选择一条成绩，点击“提交申诉”。
2. 填写申诉原因并提交。
3. 打开 `/student/appeals`，展示申诉状态为待处理。

## 6. 教师处理申诉

1. 退出学生。
2. 使用对应教师账号登录，例如 `t003 / 123456`。
3. 打开 `/teacher/appeals`。
4. 对待处理申诉选择“已通过”或“已驳回”，填写处理结果并提交。
5. 返回学生账号查看申诉状态变化。

## 7. 日志验证

1. 管理员登录。
2. 打开 `/admin/audit/score`，展示成绩修改审计。
3. 打开 `/admin/logs`，展示登录、成绩录入、成绩发布、申诉处理等操作日志。
4. 打开 `/admin/students`，点击“一键导入”，粘贴 CSV 学生数据并提交，展示批量写入 `Huangxx_Student11`。

## 8. SQL 验证

答辩时可同时展示 `sql/08_test_and_verify.sql` 的执行结果，证明视图、触发器和存储过程确实生效。
